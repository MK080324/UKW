# 引用格式规范

## 引用来源分级

### 第一级：已审批来源（docs/topic-research.md 中的来源）

这些来源已经过 QA 审批，可直接使用，不需要额外验证。

**正文内引用格式**：
```
[来源简称](URL)
```

示例：
```
安理会以13票赞成、1票反对（俄罗斯）、1票弃权（中国）的投票结果未能通过该决议草案（[UN Press SC/11319](https://press.un.org/en/2014/sc11319.doc.htm)）。
```

### 第二级：分析引用（docs/analysis-structure.md 中的分析结论）

引用分析结构中的策略分析、立场矩阵等内容时：

**正文内引用格式**：
```
（参见 analysis-structure 第X节）
```

示例：
```
中国的否决权是全场最具战略价值的筹码（参见 analysis-structure 第2.1节）。
```

### 第三级：补充来源（Writer 通过 WebSearch 新增的来源）

**正文内引用格式**：
```
[补充来源] 事实内容（[来源简称](URL)）
```

示例：
```
[补充来源] 安理会暂行议事规则第27条规定，实质性事项的决定需要九个理事国的赞成票，包括全体常任理事国的同意票（[UN Charter Art.27](https://www.un.org/en/about-us/un-charter/chapter-5)）。
```

### 第四级：背景文件引用

引用 HFIMUNC 主办方背景文件时：

**正文内引用格式**：
```
（HFIMUNC背景文件，第X页）
```

## 文档间交叉引用

当在一个文档中引用另一个文档的内容时：

**格式**：
```
（参见 D[编号] 第[章节号]章「[章节标题]」）
```

示例：
```
关于安理会表决程序的详细说明，参见 D1 第2章「会议阶段详解」。
否决权使用的策略分析，参见 D5 第3章「否决权策略」。
```

## 脚注与附录引用

### 正文引向附录
```
（详见附录[字母]）
```

示例：
```
安理会程序规则的英文原文及中文注释，详见附录A。
```

### 附录中的来源标注

附录中允许更密集的来源标注，每个事实条目后直接跟来源：

```
| 2014年3月15日 | 安理会表决 S/2014/189，13-1-1 | [UN Press SC/11319](https://press.un.org/en/2014/sc11319.doc.htm) |
```

## 来源标注密度要求

| 文档 | 最低来源引用数量 | 说明 |
|------|----------------|------|
| D1 | 安理会特殊规则章节至少 3 个权威来源 | 程序规则部分可大量使用补充来源 |
| D2 | 无硬性最低要求 | 策略建议以 analysis-structure 为基准 |
| D3 | 无硬性最低要求 | 话术以 analysis-structure 第5.3节为基准 |
| D4 | 全文至少 25 个不同来源 | 调研报告要求最高的来源密度 |
| D5 | 全文至少 15 个不同来源 | 策略文档的事实论据需要来源支撑 |
| 全项目 | 总计不少于 60 个来源引用（含重复） | Phase 4 终审检查 |

## 来源格式规范

### URL 来源
```
[来源简称](完整URL)
```
- 来源简称应简洁可识别，如 `UN Press SC/11319`、`Wikipedia: Budapest Memorandum`
- URL 必须是完整的 https:// 格式

### 文献来源（无 URL）
```
来源简称，年份
```
- 如：`HFIMUNC背景文件，2026`

### 避免
- 避免使用 `ibid.` 或 `op. cit.` 等拉丁缩写（每次引用都完整标注）
- 避免仅标注域名而不给出具体页面链接
- 避免引用已失效的链接（补充来源时注意验证链接有效性）
